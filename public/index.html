Algorithm Information

First In First Out (FIFO)

FIFO is the simplest page replacement algorithm. In this algorithm, the operating system keeps track of all pages in memory in a queue, with the oldest page at the front of the queue. When a page needs to be replaced, the oldest page is selected.

Advantages:

  • Simple to implement
  • Low overhead

Disadvantages:

  • Suffers from "Belady's Anomaly" (increasing frames can increase page faults)
  • Does not consider page usage frequency

Least Recently Used (LRU)

LRU replaces the page that has not been used for the longest period of time. This algorithm works on the idea that pages that have been heavily used in the last few instructions will likely be heavily used again in the next few.

Advantages:

  • Better performance than FIFO in most cases
  • Adapts to changing access patterns

Disadvantages:

  • More complex to implement
  • Requires hardware support for efficient implementation

Optimal (OPT or MIN)

The Optimal page replacement algorithm replaces the page that will not be used for the longest period of time. This is theoretically optimal but impossible to implement in practice as it requires future knowledge of the reference string.

Advantages:

  • Lowest possible page fault rate
  • Used as a theoretical benchmark

Disadvantages:

  • Impossible to implement in practice (requires future knowledge)
  • Can only be used for analysis and comparison

Most Recently Used (MRU)

MRU is the opposite of LRU. It replaces the most recently used page. This algorithm is based on the argument that the most recently used page should be replaced because it has just been used and may not be needed again for a while.

Advantages:

  • Works well for certain access patterns (like sequential scans)
  • Relatively simple to implement

Disadvantages:

  • Performs poorly with cyclic access patterns
  • Generally less effective than LRU
